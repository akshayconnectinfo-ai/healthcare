% ===================== 1. INTRODUCTION =====================
\section{1. Introduction}

CarePulse is a full-stack healthcare appointment management system built with \textbf{Next.js 14} (App Router), \textbf{TypeScript}, and \textbf{Appwrite} as the backend-as-a-service. The system enables patients to register, book appointments with physicians, and receive SMS notifications. An admin dashboard provides oversight of all appointments with schedule/cancel capabilities.

\subsection{1.1 System Modules Under Test}
\begin{enumerate}[leftmargin=*, noitemsep]
    \item \textbf{M1 -- Patient Registration Module:} User creation (\texttt{PatientForm}), detailed patient registration (\texttt{RegisterForm}), identification document upload, Zod schema validation.
    \item \textbf{M2 -- Appointment Management Module:} Appointment creation, scheduling, cancellation (\texttt{AppointmentForm}), physician selection, date/time scheduling.
    \item \textbf{M3 -- Admin Dashboard Module:} Passkey-protected admin access (\texttt{PasskeyModal}), statistical overview cards (\texttt{StatCard}), data table with appointment listing (\texttt{DataTable}).
    \item \textbf{M4 -- Notification \& Backend Module:} SMS notifications via Appwrite Messaging, Appwrite database CRUD operations, file storage for identification documents.
\end{enumerate}

% ===================== 4. SYSTEM & USER TESTING PLAN =====================
\section{4. System Testing \& User Testing Plan}

\subsection{4.1 System Testing}

System testing validates complete end-to-end workflows across all integrated modules:

\begin{enumerate}[leftmargin=*, noitemsep]
    \item \textbf{Workflow 1 -- Patient Onboarding:} Open landing page $\rightarrow$ fill \texttt{PatientForm} $\rightarrow$ submit $\rightarrow$ verify user creation $\rightarrow$ complete \texttt{RegisterForm} with all medical details and ID upload $\rightarrow$ verify patient document in Appwrite.
    \item \textbf{Workflow 2 -- Appointment Lifecycle:} Logged-in patient $\rightarrow$ create appointment via \texttt{AppointmentForm} $\rightarrow$ verify pending status $\rightarrow$ admin schedules $\rightarrow$ verify SMS received $\rightarrow$ admin cancels $\rightarrow$ verify cancellation SMS.
    \item \textbf{Workflow 3 -- Admin Operations:} Navigate to \texttt{/admin} $\rightarrow$ enter passkey via OTP modal $\rightarrow$ verify stat cards reflect real data $\rightarrow$ sort/filter appointments in DataTable $\rightarrow$ schedule and cancel appointments.
\end{enumerate}

\textbf{Environment:} Staging Appwrite instance with seeded test data. Browser testing on Chrome, Firefox, and Safari (desktop \& mobile viewports).

\subsection{4.2 User Acceptance Testing (UAT)}

\begin{tabular}{@{}p{3.5cm}p{10.5cm}@{}}
\toprule
\textbf{UAT Criterion} & \textbf{Evaluation} \\
\midrule
Usability & Forms are intuitive; error messages are clear and contextual; loading states (\texttt{SubmitButton}) provide feedback. \\
Responsiveness & Layout adapts correctly on mobile (375px), tablet (768px), and desktop (1440px) breakpoints. \\
Accessibility & All form fields have labels; keyboard navigation works for modals and forms; color contrast meets WCAG AA. \\
Data Integrity & All patient and appointment data persists correctly in Appwrite; no data truncation beyond expected 32-char limits. \\
Notification Accuracy & SMS messages contain correct appointment details, physician names, and formatted date/time for the user's timezone. \\
\bottomrule
\end{tabular}
